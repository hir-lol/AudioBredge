# Команды и структура сообщений
---
Сообщения должны быть в **json** формате, пример :
``` json
{
    "type": "VOLUME",
    "value": 110 
}
```
**type** - сама команда,

**value** - дополнительная информация

## Команды от Клиента к Серверу:
- VOLUME - обновление громкости <details> <summary>?</summary> Сервер выводит уведомление в трей и увеличивает громкость звука если значение больше 100 
- STOP - остановка отправления звука и закрытие соеденения
- SEND_TRAY - Отправка команды трею
<details> <summary> Пример команды для трея </summary>

``` json
{
    "type": "SEND_TRAY",
    "value": {
        "type":"NOTIFY",
        "value":"Тестовое уведомление"
    }
}
```

</details>


## Команды от Сервера к Клиенту:
- VOLUME - Обновление громкости от трея 
- PING - Проверка клиента на активность, клиент должен на него ответить 
- RESTART - Перезапуск TCP и UDP соеденения

<details> <summary> Пример ответа клиента на пинг </summary>

``` json
{
    "type": "PONG"
}
```

</details>

<details> <summary> Пример пакета обновения громкости  </summary>

``` json
{
    "type": "VOLUME",
    "value": 50
}
```

</details>

## Команды трея 
- NOTIFY - Вывод уведомления
- VOLUME - Изменения громкости без вывода уведомления
